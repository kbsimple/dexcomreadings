  <action>Fix all lines in `dexcom_readings.py` that exceed 80 characters. Google Python Style Guide mandates a maximum line length of 80 characters.

**Lines that need fixing (after previous plan changes, line numbers may shift):**

1. **f-strings spanning multiple lines** - break at natural points:
   ```python
   # BEFORE:
   logging.info(f"Connecting to Dexcom Share for user {DEXCOM_USERNAME} "
         f"in region {DEXCOM_REGION}...")

   # AFTER:
   logging.info(
       f"Connecting to Dexcom Share for user {DEXCOM_USERNAME} "
       f"in region {DEXCOM_REGION}..."
   )
   ```

2. **Long log messages** - use implicit string concatenation or break parameters:
   ```python
   # BEFORE:
   logging.warning("NIGHTSCOUT_URL or NIGHTSCOUT_API_SECRET not set. "
         "Nightscout upload will be skipped.")

   # AFTER:
   logging.warning(
       "NIGHTSCOUT_URL or NIGHTSCOUT_API_SECRET not set. "
       "Nightscout upload will be skipped."
   )
   ```

3. **Long f-strings in main loop** - break at natural points:
   ```python
   # BEFORE:
   logging.info(f"{check_timestamp_utc.isoformat()}: New reading! Value: {current_bg.value} mg/dL "
         f"({current_bg.trend_description}), Time: "
         f"{current_glucose_datetime.isoformat()}")

   # AFTER:
   logging.info(
       f"{check_timestamp_utc.isoformat()}: New reading! Value: "
       f"{current_bg.value} mg/dL ({current_bg.trend_description}), "
       f"Time: {current_glucose_datetime.isoformat()}"
   )
   ```

4. **Long comment lines** - break comments:
   ```python
   # BEFORE:
   # Optional: Specify if outside the US, e.g., "ous" for "outside US" or "jp" for Japan. Default is "us".

   # AFTER:
   # Optional: Specify if outside the US, e.g., "ous" for "outside US"
   # or "jp" for Japan. Default is "us".
   ```

**Strategies:**
- Break at operators, commas, or parentheses
- Use implicit string concatenation for adjacent string literals
- For function calls with long arguments, put each argument on its own line
- For logging statements, break after the opening parenthesis

**Do NOT use backslash continuation** (against Google Style Guide).

**Create .pylintrc with the following content:**

```ini
[MASTER]
# Python code to execute before pylint runs
init-hook=''

# Use multiple processes to speed up pylint
jobs=1

[MESSAGES CONTROL]
# Disable specific warnings that are acceptable for this project
disable=
    missing-module-docstring,
    too-few-public-methods,
    broad-exception-caught,
    global-statement

[FORMAT]
# Maximum line length
max-line-length=80

# Regexp for a line that creates a function call
function-rgx=[a-z_][a-z0-9_]{2,40}$

# Regexp for a line that creates a variable
variable-rgx=[a-z_][a-z0-9_]{1,30}$

# Regexp for a line that creates a constant
const-rgx=(([A-Z_][A-Z0-9_]*)|(__.*__))$

[TYPECHECK]
# List of module names for which member attributes should not be checked
ignored-modules=pydexcom

[BASIC]
# Good variable names which should always be accepted
good-names=i, j, k, e, f, _

[SIMILARITIES]
# Minimum lines number of a similarity
min-similarity-lines=4

# Ignore comments when computing similarities
ignore-comments=yes

# Ignore docstrings when computing similarities
ignore-docstrings=yes

# Ignore imports when computing similarities
ignore-imports=yes
```

**Rationale for suppressions:**
- `missing-module-docstring`: Will be addressed after this plan; suppressing temporarily
- `too-few-public-methods`: Not applicable for this script-style module
- `broad-exception-caught`: The code catches Exception for API error handling (acceptable for network operations)
- `global-statement`: Was removed in Plan 02; keeping suppression in case of future false positive
- `ignored-modules=pydexcom`: Third-party library, type checking may not be available

**Note:** If `missing-module-docstring` was added in Plan 01, remove it from the disable list here. The docstring should now exist.</action>
